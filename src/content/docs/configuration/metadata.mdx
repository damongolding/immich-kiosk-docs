### Switch to years after

Switch to years after a certain age.

<Settings config="age_switch_to_years_after" env="KIOSK_AGE_SWITCH_TO_YEARS_AFTER" value="number" defaultValue="1" />

#### Example with `age_switch_to_years_after: 10`

| 9 years and 4 months | 10 years 2 months |
| -------------------- | ----------------- |
| `NAME (9y 4m)`       | `NAME (10)`       |


### Always display year unit

<Settings config="show_age_year_unit" env="KIOSK_SHOW_AGE_YEAR_UNIT" value="bool" defaultValue="false" />

Set to true if you would prefer to always display the year unit e.g. `NAME (24y)`.

## Image Date Format

The format of the image date. default is day/month/year. See date format for more information.

<Settings config="image_date_format" env="KIOSK_IMAGE_DATE_FORMAT" value="string" defaultValue="DD/MM/YYYY" />

### Date format

<Aside type="note">Some characters, such as `/` and `:` are not allowed in URL params. So while you can set the date layout via URL params, I would suggest setting them via `config.yaml` or environment variables.</Aside>

You can use the below values to create your preferred date layout.

| **Value** | **Example output** |
| --------- | ------------------ |
| YYYY      | 2024               |
| YY        | 24                 |
| MMMM      | August             |
| MMM       | Aug                |
| MM        | 08                 |
| M         | 8                  |
| DDDD      | Monday             |
| DDD       | Mon                |
| DD        | 04                 |
| D         | 4                  |

### Date layout examples

These examples assume that today's date is the 22nd of August 2024.

- `YYYY-MM-DD` => `2024-08-22`
- `YYYY/MM/DD` => `2024/08/22`
- `YYYY:MM:DD` => `2024:08:22`
- `YYYY MM DD` => `2024 08 22`
- `YYYY MMM (DDD)` => `2024 Aug (Thur)`
- `DDDD DD MMMM YYYY` => `Thursday 22 August 2024`
