<Aside type="note">Frameo devices tend to be low-powered with limited resources. It is recommended to disable any [image effects](/configuration/image-effects/) and depending on your device you may need to enable [optimize_images](/configuration/behavior/#optimize-images).</Aside>

## Prerequisites

- [adb](https://developer.android.com/tools/releases/platform-tools) downloaded and installed
- ADB access enabled on the frame: `Settings -> About -> Enable Beta Program`
- A USB-C to USB-A cable may be needed - some frames do not have TCP/IP enabled by default

## Connecting via USB

<Aside type="tip">
    Some Frameo frames only allow one ADB connection at a time. If connecting multiple devices, each will need to go through this process.
</Aside>

<Steps>
    1. Connect the Frameo frame to your computer via USB

    2. List connected devices:
       ```sh
       adb devices
       ```

    3. If the device shows as `unauthorized`, go to `Settings -> About -> Enable Beta Program` on the frame and toggle it off and back on. A reboot may also be needed.

    4. Once the device shows as `device`, enable TCP/IP for wireless ADB:
        ```sh
        adb tcpip 5555
        ```

</Steps>

## Connecting wirelessly

Once TCP/IP is enabled (see above), you can connect without USB:

```sh
adb connect <device_ip>:5555
```

<Aside type="caution">
    Network ADB resets after reboot. You will need to re-enable it via USB after each restart, or use a tool like Tasker to automate this.
</Aside>

### Making ADB persistent (optional)

If you want ADB to survive reboots without needing to toggle Beta Program each time:

```sh
# Set persistent USB configuration for ADB
adb shell "setprop persist.sys.usb.config adb"

# Ensure ADB and developer settings stay enabled
adb shell "settings put global adb_enabled 1"
adb shell "settings put global development_settings_enabled 1"
```

Verify it worked:

```sh
adb shell getprop persist.sys.usb.config
# Should output: adb
```

Test after reboot:

```sh
adb reboot
# Wait for device to fully boot (30-60 seconds)
adb devices
```

## Updating the system WebView

<Aside type="danger">
    Not all Frameo devices are the same, hardware and software versions may differ and these steps may not work for every device. Proceed at your own risk. We accept no responsibility for any damage or issues that may result.
</Aside>

<Aside type="caution">
    Check your current WebView version before proceeding: 
    ```sh 
    adb shell dumpsys webviewupdate
    ``` 
    If you have **WebView v101 or newer**, skip this section - your device is already compatible.
</Aside>

<Aside type="tip">
    The WebView APK path may vary by device. To find it: 
    ```sh 
    adb shell pm path com.android.webview
    ```
</Aside>

Older Frameo devices ship with a system WebView that is not compatible with Kiosk. To update it:

<Steps>

    1. Download the 106 WebView apk from [here](https://www.apkmirror.com/apk/lineageos/android-system-webview-2/android-system-webview-2-106-0-5249-126-release/android-system-webview-106-0-5249-126-12-android-apk-download/)

    2. Connect to your device with ADB (see [connection options](#connecting-to-the-frameo-frame-with-adb) above - USB or network)

    3. Push the WebView apk to the device
       ```sh
       adb push path/to/downloaded/webview.apk /sdcard/
       ```
    4. Enter root
       ```sh
       adb shell su
       ```

    5. Backup the original WebView apk
       ```sh
       mount -o remount,rw /system && cp /system/app/webview/webview.apk /system/app/webview/webview.apk.bak
       ```

    6. Remove the existing WebView oat files
       ```sh
       mount -o remount,rw /system && rm -rf /system/app/webview/oat
       ```

    7. Copy the new WebView apk to the device
       ```sh
       mount -o remount,rw /system && cp /sdcard/webview.apk /system/app/webview/webview.apk
       ```

    8. Exit root
       ```sh
       exit
       ```

    9. Reboot the device
        ```sh
        adb reboot
        ```

</Steps>

## Displaying Kiosk on your Frameo

Choose one of the apps below - you do not need to install both.

<Tabs syncKey="display_app">
    <TabItem label="Fully Kiosk Browser">

        <Aside type="tip">
            Setting `sleep_dim_screen=true` will automatically dim the screen when the device enters sleep mode.
        </Aside>

        <Steps>
            1. Download the latest Fully Kiosk Browser APK from [here](https://www.fully-kiosk.com/files/2025/01/Fully-Kiosk-Browser-v1.57.1.apk)

            2. Install it:
                ```sh
                adb install path/to/downloaded/Fully-Kiosk-Browser.apk
                ```

            3. Launch it:
                ```sh
                db shell am start -n de.ozerov.fully/.MainActivity
                ```
        </Steps>

    </TabItem>

    <TabItem label="ImmichFrame">

        <Aside type="tip">
            Setting `sleep_dim_screen=true` will automatically dim the screen when the device enters sleep mode.
        </Aside>

        Although Kiosk doesn't have its own Android app, the ImmichFrame team has built a native Android app that works with Kiosk.

        <Aside type="note">
            ImmichFrame has an option to set an auth secret in its settings. This is passed to Kiosk as the [password](/configuration/additional-options/#password).
        </Aside>

        <Steps>
            1. Download the latest ImmichFrame APK from [here](https://github.com/immichFrame/ImmichFrame_Android/releases)

            2. Install it:
                ```sh
                adb install path/to/downloaded/immichframe.apk
                ```

            3. Launch it and configure your Kiosk server URL:
                ```sh
                adb shell am start -n com.immichframe.immichframe/.MainActivity
                ```
        </Steps>

        #### Uninstalling ImmichFrame

        <Steps>
            1. Uninstall the app:
                ```sh
                adb uninstall com.immichframe.immichframe
                ```

            2. Re-enable the Frameo app:
                ```sh
                pm enable net.frameo.frame
                ```
        </Steps>

    </TabItem>

</Tabs>

## Disable the Frameo app

<Tabs syncKey="disable_method">
    <TabItem label="Without root (Android 8+)">
        ```sh
        adb shell pm disable-user --user 0 net.frameo.frame
        ```
    </TabItem>

    <TabItem label="With root (Android 6/7)">
        <Steps>
            1. Enter root:
                ```sh
                adb shell su
                ```

            2. Disable the Frameo app:
                ```sh
                pm disable net.frameo.frame
                ```

            3. Exit root:
                ```sh
                exit
                ```
        </Steps>
    </TabItem>

</Tabs>

## Troubleshooting

### Device loses WiFi after disabling the Frameo app

The Frameo setup wizard configures WiFi within the Frameo app only - not in Android system settings. When the Frameo app is disabled, that WiFi config is no longer active.

**Fix:** Before disabling the Frameo app, connect to your WiFi network via Android system settings (`Network & Internet -> WiFi`), not through the Frameo wizard.

If you have already disabled the Frameo app and lost connectivity:

<Steps>
    1. Re-enable the Frameo app:
        ```sh
        adb shell pm enable net.frameo.frame
        ```

    2. Open Android WiFi settings and connect to your network

    3. Disable the Frameo app again once connectivity is confirmed

</Steps>

### ADB shows "unauthorized"

Go to `Settings -> About -> Enable Beta Program` on the frame and toggle it off and back on. Accept the authorisation prompt on the device screen, then retry:

```sh
adb devices
```

### ImmichFrame not launching on boot

Set it as the default home launcher:

```sh
adb shell cmd package set-home-activity com.immichframe.immichframe/.MainActivity
```

### Restoring the original Frameo experience

```sh
# Re-enable the Frameo app
adb shell pm enable net.frameo.frame

# Set it as default launcher
adb shell cmd package set-home-activity net.frameo.frame/.MainActivity

# Optionally uninstall ImmichFrame
adb uninstall com.immichframe.immichframe
```
